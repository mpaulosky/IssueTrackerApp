## Summary

**v0.7.0 is now live.** This release is an engineering quality milestone — no new user-facing features, but the codebase is now measurably safer and more maintainable. A three-sprint code coverage initiative brings line coverage to **82.5%** with a CI gate that fails any PR dropping below 80%. Alongside that, a full UI polish pass standardizes Tailwind colour usage, button classes, and table styles across the entire application. A real production race condition in `InMemoryBulkOperationQueue` was discovered and fixed in the process.

## What's New

### 🧪 Code Coverage Improvement (Milestone #9)

**Three sprints, 16 issues, 22 new test files, +6,871 lines of tests.**

#### Sprint 1 — Infrastructure & Critical Gaps (#209)

The foundation: coverage tooling wired into CI and the most critical untested areas addressed.

- **CI Coverage Gate**: `ReportGenerator` added to `squad-test.yml`. Every PR now fails if line coverage drops below 80%. HTML coverage reports are published as CI artifacts.
- **Coverage Badge**: `README.md` updated with a live Codecov badge linked to the branch report.
- **`GetUserDashboardQueryHandler` tests** (15 tests): Full unit test coverage of the Dashboard domain query handler — loading state, empty data, error propagation.
- **`VotingService` tests** (18 tests): Upvote, downvote, toggle, duplicate-vote idempotency, and repository interaction.
- **`VoteEndpoints` integration tests** (17 tests): `POST /api/issues/{id}/vote` and `GET /api/issues/{id}/vote` — authenticated, unauthenticated, and edge cases.
- **`Dashboard.razor` bUnit tests** (22 tests): Component rendering, loading skeleton, SignalR connection indicator, empty-state and error-state branches.

#### Sprint 2 — Service & Endpoint Coverage (#212)

Closing gaps in web services, integrations, and infrastructure (+98 tests).

- **`LocalFileStorageService` tests** (47 tests): File upload, download, delete, path sanitization, and size-limit enforcement.
- **`SendGridEmailService` tests** (51 tests): Mocked HTTP client tests covering success, API error, network failure, and retry behaviour.
- **`LabelEndpoints` integration tests** (28 tests): `GET /api/labels/suggestions` — prefix filtering, empty results, authentication gate.
- **`IssueHub` SignalR integration tests** (38 tests): Hub connection lifecycle, `ReceiveIssueUpdate` broadcast, and reconnection handling.
- **`ObjectIdJsonConverter` unit tests** (15 tests): Round-trip serialization for all valid ObjectId formats, null handling, and invalid input rejection.
- **`ServiceDefaults` health endpoint tests** (10 tests): `/health` and `/alive` endpoint responses under normal and degraded states.
- **Race condition fix** in `InMemoryBulkOperationQueue.QueueAsync`: The background service could read an enqueued item and set its status to `Failed` *before* `QueueAsync` set the initial `Queued` status — causing a test flake that was actually a real production bug. Fixed by setting `Queued` status **before** writing to the channel.

#### Sprint 3 — Component & Documentation Coverage (#213)

Full bUnit coverage of UI components and coverage standards documented (+6,871 lines across 13 test files).

- **Analytics page bUnit tests** (467 lines): Chart data binding, loading/error states, metric card rendering.
- **Dashboard bUnit tests** (494 lines): Stream rendering, quick-action cards, SignalR status.
- **Shared component tests**: `Pagination` (306), `SearchInput` (227), `StatusBadge` (348), `DeleteConfirmationModal` (283).
- **Theme component tests**: `ThemeProvider` (270), `ThemeColorDropdown` (148), `ThemeBrightnessToggle` (131).
- **Bulk/Issues UI tests**: `AttachmentCard` (301), `BulkConfirmationModal` (241), `BulkProgressIndicator` (301), `UndoToast` (242).
- **CONTRIBUTING.md** — new *Code Coverage Standards* section: how to run coverage locally (`dotnet test --collect:"XPlat Code Coverage"`), what the 80% gate enforces, and how to read the HTML artifact.

### 🎨 UI Polish & Styling Standardization (#188, #189, #191)

Three PRs that swept through the entire UI for consistency.

**Button & link standardization (#188)**

All interactive elements now use the `.btn` base class before variant modifiers. Shared hover, focus, and transition properties were extracted into the base class, eliminating ~40 lines of duplicated CSS.

```css
/* Before: each variant repeated shared properties */
.btn-primary { padding: ...; border-radius: ...; transition: ...; }
.btn-secondary { padding: ...; border-radius: ...; transition: ...; }

/* After: shared properties in base, variant only adds what differs */
.btn { padding: ...; border-radius: ...; transition: ...; }
.btn-primary { background-color: ...; color: ...; }
.btn-secondary { background-color: ...; color: ...; }
```

**Admin layout colour refactor (#189)**

Replaced all `-gray-` Tailwind shade suffixes with `-primary-` across 38 Razor files (~270 occurrences). This ensures the admin layout respects the active colour theme rather than staying locked to a neutral gray palette.

**Table styling standardization (#191)**

`UserListTable.razor` was the gold standard. All admin and user-facing tables now share the same header (`bg-primary-700` + `text-primary-300`), row hover, and responsive overflow patterns.
