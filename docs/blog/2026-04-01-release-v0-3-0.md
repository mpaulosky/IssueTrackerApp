## Summary

**v0.3.0 is now live.** This release focuses on polish and developer experience: the Dashboard Quick Actions layout has been corrected to stack vertically for better mobile responsiveness, Redis caching infrastructure is now integrated for performance optimization, issues can now be restored after deletion via a new `RestoreIssueCommand`, and the Issue model gains full assignee field support. Together, these changes solidify the v0.3 Polish Sprint and prepare the foundation for richer issue workflows in future releases.

## What's New

### Features

The v0.3.0 feature set emphasizes **user experience refinement, data recovery, and team workflows**:

- **Assignee Field on Issues** (#122): The Issue model now includes full `AssignedTo` support. Users can assign issues to team members, enabling collaborative workflows and responsibility tracking across the project.
- **Restore Issue Command** (#121): A new `RestoreIssueCommand` allows soft-deleted issues to be recovered. Combined with the existing soft-delete pattern, this gives teams a safety net for accidental deletions.
- **Redis Caching Layer** (#123): Redis is now provisioned and integrated into the Aspire local dev environment. The caching infrastructure is ready for analytics, dashboard, and high-frequency query optimization.

### Bug Fixes

The Dashboard Quick Actions component has been refined for accessibility and mobile responsiveness:

- **Dashboard Quick Actions Vertical Stack** (#120): Fixed the Quick Actions layout to stack vertically instead of horizontally on smaller screens. This improves mobile usability and brings the dashboard in line with responsive design standards.

### Developer Tooling & Devops

Release automation and session logging have been formalized:

- **Release Workflow Documentation** (#118, #119): Added session logs and orchestration documentation for the v0.2.0 release. Every release now includes formal decision logs and squad session records, enabling future contributors to understand the context and reasoning behind each release.

### Assignee Field Design

Assignees are stored as `ObjectId` references to `ApplicationUser`. The Issue model now supports team-based workflows:

```csharp

```

### Redis Provisioning in Aspire

Redis is now spun up alongside MongoDB in the AppHost for local development. This sets the stage for distributed caching, real-time event subscriptions, and performance optimizations in future releases.
