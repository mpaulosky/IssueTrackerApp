## Summary

**v0.2.0 is now live.** This release represents a major quality and documentation push: a neutral color migration for the UI, complete GitHub Pages deployment automation, all skipped tests audited and re-enabled, formal peer review gates, and a new blog where the squad documents every release. Under the hood, we've strengthened E2E test coverage, unified developer tooling, and formalized the path to production.

## What's New

### Features

The v0.2.0 feature set focuses on **UI cohesion, navigation simplification, and accessibility**:

- **UI Refresh** (#61): Migrated from custom color schemes to a neutral palette with system color detection. The new theme system is faster, more accessible, and integrates with browser dark mode automatically.
- **Consolidated Navigation** (#71, #72): Moved all header navigation into a single `NavMenuComponent` and repositioned `ThemeToggle` into Shared components. The home page is now cleaner; mobile hamburger menu is restored (#65).
- **Access Denied Page** (#83): Added `/Account/AccessDenied` Blazor page for clearer auth rejection handling.
- **UI Polish Sprints** (#99, #101, #102): Unified text sizes across footer, navigation headers, and the SignalR status indicator. Dark mode text colors now pass accessibility standards. Dashboard and nav footer refined for consistency.

### Bug Fixes

We fixed 13 end-to-end test failures and resolved auth/admin page crashes, plus critical build system issues:

- **E2E Test Failures** (#86): Diagnosed and fixed 13 Playwright test failures; Issues page bugs and accessibility violations resolved.
- **Auth0 Role Claims** (#100): Resolved role claim remapping in Auth0; fixed `AdminPageLayout` crash when users lack admin roles.
- **MongoDB Connection String Fallback** (#60): Connection string now properly falls back from `MongoDB:ConnectionString` to `ConnectionStrings:mongodb`, critical for Aspire + Docker environments.
- **Tailwind CSS Loading** (#65): Restored Tailwind CSS loading in the build pipeline and added mobile hamburger menu.
- **Build Info Git Stderr** (#59): Suppressed noisy git stderr in `BuildInfo` generation; restructured Tailwind CSS build step.
- **Pre-Push Gate Hardening** (#106): Strengthened pre-push hook to catch more issues before commit; synced README with actual requirements.

### Testing

All 8 previously skipped tests are now unskipped and passing. We introduced formal validation pipelines and expanded test coverage significantly:

- **Unskip All Tests** (#115): Audited 8 skipped tests; all skip reasons remain valid, but tests are now re-enabled with added `ValidationBehavior` pipeline support in MediatR.
- **Audit Skipped Tests** (#62): Completed a full audit of test skips—every skip reason is still valid, but code changes allow them to run safely now.
- **AppHost Tests + Playwright E2E** (#76): 3 Aspire integration tests + 29 Playwright E2E tests across 8 files; full stack validation without Auth0 dependency during test runs.
- **Stronger Assertions & Descriptive Timeouts** (#84): Improved test reliability with better error messages and consistent timeout handling.

### Developer Tooling & DevOps

The squad formalized tooling, workflows, and deployment processes. This release marks the beginning of a **documented, gated, and auditable workflow**:

- **GitVersion Integration** (#116): Added GitVersion for semantic versioning; release workflows and milestone blog trigger configured.
- **Formal PR Review Process** (#82, #103): Squad established a PR review gate; all PRs now require peer review before merge. Pages scope limited to `docs/blog/` for blog safety.
- **GitHub Actions Workflows** (#81): Automated GitHub Pages deployment workflow — every blog post and docs change now auto-deploys.
- **Hook Template + Pre-Push Gate** (#113, #114): Committed hook template; `install-hooks.sh` script provided. Pre-push gate enforces linting, tests, and build checks locally.
- **Squad SDK Scaffold** (#74): Formalized agent charters, routing rules, and team structure at `.squad/`. UI color improvements and `ToastContainer` thread-safety fix applied.
- **Contributing Guide** (#114): Created `CONTRIBUTING.md` with comprehensive pre-push gate requirements, branch naming, and peer review expectations.
- **Latest Release Badge** (#117): Added Latest Release badge to README for quick visibility.
- **Dependency Updates** (#87, #73): Bumped all-actions group (5 updates) and picomatch (security patch). Scribe charter updated (#63) to commit on feature branches only.

### Documentation

The squad codified squad roles, blog process, and project decisions:

## Architecture Highlights

### Vertical Slice with MediatR
Commands and Queries live in `src/Domain/Features/*`. Every feature is a self-contained unit:

```
Domain/
  Features/
    Issues/
      Commands/ → CreateIssueCommand, UpdateIssueCommand, DeleteIssueCommand
      Queries/ → GetIssuesQuery, GetIssueByIdQuery
      Validators/ → CreateIssueValidator
      Events/ → IssueCreatedEvent, IssueUpdatedEvent
```

### GitHub Pages + Plain Markdown
No Jekyll config needed. Blog lives at `docs/blog/`; GitHub auto-renders Markdown. Every release gets a post.

### Aspire Orchestration
Local dev runs via `dotnet run --project src/AppHost/AppHost.csproj`. AppHost wires up web, MongoDB, Redis, and runs health checks before accepting requests.

## How to Upgrade

1. **From v0.1.0 → v0.2.0**: No breaking changes. Git pull, `dotnet restore`, `dotnet build`, and you're done.
2. **New blog content**: Check out `docs/blog/` for documentation on every major change.
3. **Pre-push hooks**: Run `./scripts/install-hooks.sh` to install the pre-push gate (recommended for contributors).
